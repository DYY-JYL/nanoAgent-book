# 从零开始理解大模型（番外篇六）：什么是算子？大模型里的“最小施工单位”

> **「从零开始理解大模型」番外篇** —— 正篇十篇讲原理，番外篇回答那些“懂了原理之后自然会好奇”的问题。

你可能在技术文章或新闻里见过“算子”这个词：

- “XX 框架优化了 Attention 算子，推理速度提升 30%”
- “该芯片支持 200+ 种 AI 算子”
- “FlashAttention 是一种高效的注意力算子实现”

“算子”听起来很学术，但其实概念很简单。读完正篇的你已经见过了所有常见算子——只是当时不叫这个名字。

-----

## 一、先说结论

| 你以为的 | 实际的 |
|-----------|--------------------------------|
| 算子是一个很高深的概念 | 算子就是“一个确定的计算步骤”，比如矩阵乘法、Softmax |
| 大模型的计算很复杂 | 拆到底就是十几种算子的排列组合，同一类算子会被重复调用成百上千次 |
| 算子和我没关系 | 你用的每一个大模型，推理速度和成本都取决于算子的实现效率 |

一句话版本：**算子（Operator）是大模型计算中的“最小施工单位”——每个算子代表一个明确的输入→计算规则→输出的完整步骤。虽然在底层可以进一步融合或优化，但对模型开发者来说，它就是最小的独立计算单位。**

-----

## 二、你已经见过的所有算子

正篇十篇里出现的每一步计算，都对应一个算子。我们来对一遍：

### 2.1 Embedding 查表（第三篇）

```text
输入: token ID = 10449
操作: 在 50257×768 的表里查第 10449 行
输出: 一个 768 维向量
```

这就是一个算子——**Embedding 算子**。输入是整数索引，输出是向量，操作是查表。

### 2.2 矩阵乘法（第四篇、第五篇）

```text
输入: 向量 x (768维), 权重矩阵 W (768×768)
操作: x × W
输出: 新向量 (768维)
```

这是最重要的算子——**MatMul（矩阵乘法）算子**。Q/K/V 投影、FFN 的 W₁·x 和 W₂·x、LM Head 的输出映射，全都是它。番外篇二讲过，大模型的计算量大部分花在矩阵乘法上。

### 2.3 Softmax（第四篇）

```text
输入: 一组分数 [2.0, 1.0, 0.5, 0.1]
操作: 对每个数取 e 的指数，再除以总和
输出: 概率分布 [0.52, 0.19, 0.12, 0.08]（加起来 = 1）
```

**Softmax 算子**。在 Attention 打分后把分数变成概率，在最终输出时把 logits 变成词的概率。

### 2.4 LayerNorm（第五篇）

```text
输入: 向量 [100.0, 0.001, -50.0, 25.0]
操作: 减均值，除标准差，乘 γ 加 β
输出: 归一化向量 [1.45, -0.34, -1.23, 0.11]
```

**LayerNorm 算子**。每层 Transformer 前做一次，稳定数值。

### 2.5 GELU 激活（第五篇）

```text
输入: 一组数字 [1.5, -0.3, 2.0, -1.0]
操作: 对每个数字应用 GELU 函数（正数基本保留，负数被压低）
输出: [1.47, -0.11, 1.99, -0.16]
```

**GELU 算子**。FFN 中间的激活函数。

### 2.6 Add（加法）（第五篇）

```text
输入: 向量 A, 向量 B
操作: 对应位置相加
输出: A + B
```

**Add 算子**。残差连接就是它——输入 + Attention 输出、输入 + FFN 输出。

### 2.7 还有一些

| 算子 | 出现在哪 | 做什么 |
|---------|------------|---------------|
| Transpose | Attention | 矩阵转置（Kᵀ） |
| Scale | Attention | 除以 √d |
| Mask | Attention | 因果掩码，把未来位置设为 -∞ |
| Concat | 多头 Attention | 把多个头的输出拼接 |
| Reshape | 到处都有 | 改变张量形状（不改数据） |

## 三、一次前向传播 = 一串算子按顺序执行

第七篇讲过，生成一个 token 需要跑一次完整的前向传播。现在你可以把它理解为一条**算子流水线**：

```text
Token ID
  │
  ▼ Embedding 算子 ──→ 向量
  ▼ Add 算子 ──→ 向量 + 位置编码
  │
  │ ┌─── 以下重复 32 层 ───┐
  │ │                      │
  ▼ │ LayerNorm 算子        │
  ▼ │ MatMul 算子 ×3 (QKV)  │
  ▼ │ MatMul 算子 (Q·Kᵀ)   │
  ▼ │ Scale 算子 (÷√d)     │
  ▼ │ Mask 算子             │
  ▼ │ Softmax 算子          │
  ▼ │ MatMul 算子 (·V)      │
  ▼ │ Concat 算子           │
  ▼ │ MatMul 算子 (输出投影) │
  ▼ │ Add 算子 (残差)       │
  ▼ │ LayerNorm 算子        │
  ▼ │ MatMul 算子 (W₁)      │
  ▼ │ GELU 算子             │
  ▼ │ MatMul 算子 (W₂)      │
  ▼ │ Add 算子 (残差)       │
  │ │                      │
  │ └──────────────────────┘
  │
  ▼ LayerNorm 算子（最终）
  ▼ MatMul 算子（LM Head）
  ▼ Softmax 算子
  │
  ▼ 概率分布 → 选一个 token
```

数一下：一层 Transformer 里大约有 **15 个算子**。32 层就是约 480 个算子，加上首尾的 Embedding 和 LM Head，一次前向传播大约执行 **500 个算子**。

> 注：这是在未做大量算子融合前的粗略计数。经过 FlashAttention 等优化后，多个算子会被合并成一个 kernel，实际执行的 kernel 数量会大幅减少。

每个算子做的事情都很简单——矩阵乘法、加法、取指数、查表……但 500 个简单操作串在一起，就构成了大模型的全部“智能”。

## 四、为什么要知道“算子”这个概念

### 4.1 它是框架和硬件之间的“合同”

当你写 `model(input)` 的时候，PyTorch 把你的模型拆成一个个算子，然后交给 GPU 执行。每个算子对应 GPU 上的一段优化过的计算代码（叫 **kernel**）。

```text
你的代码:     model(input)
  ↓
PyTorch:     拆成算子序列 [Embedding, MatMul, Softmax, ...]
  ↓
CUDA kernel: 每个算子对应 GPU 上的一段优化代码
  ↓
GPU 硬件:    执行
```

算子就是**软件和硬件之间的接口**。框架说“我要做一个 MatMul”，GPU 说“好的，我有一个高度优化的 MatMul kernel”。两边通过算子这个“合同”来对接。

### 4.2 不同硬件支持的算子不同

NVIDIA GPU 通过 CUDA 提供了几百种优化好的算子 kernel。其他芯片（华为昇腾、苹果 M 系列、AMD GPU）也各自实现了自己的算子库。

当你看到“XX 芯片对大模型的兼容性”时，核心问题就是：**这颗芯片的算子库是否覆盖了大模型需要的所有算子，以及每个算子的效率如何。**

### 4.3 算子优化 = 推理变快

同一个算子可以有不同的实现方式，效率差别很大。

最著名的例子是 **FlashAttention**——它不是一种新的 Attention 机制（数学上和第四篇讲的完全一样），而是把 Attention 涉及的多个算子（MatMul + Scale + Mask + Softmax + MatMul）**合并成一个高效算子**，减少了数据在 GPU 显存和计算核心之间的搬运次数。

```text
标准实现:  MatMul → 存到显存 → Scale → 存到显存 → Mask → 存到显存 → Softmax → 存到显存 → MatMul
           ↑ 每步都要把数据搬进搬出显存，搬运很慢

FlashAttention: [MatMul + Scale + Mask + Softmax + MatMul] 一口气算完
                ↑ 数据留在芯片的高速缓存里，不来回搬
```

数学结果完全一样，但速度快了 2-4 倍。这就是“算子融合”（Operator Fusion）——把多个小算子合并成一个大算子，减少中间的数据搬运。

## 五、算子的完整图景

把正篇和番外篇的知识串起来：

```text
你问模型一个问题
  │
  ▼ 分词（第二篇）——不是算子，是预处理
  │
  ▼ 约 500 个算子按顺序执行（本篇）
  │  ├── 大部分计算量在 MatMul 算子（番外篇二：GPU 擅长的矩阵乘法）
  │  ├── Softmax、LayerNorm 等算子计算量小但不可少
  │  └── 算子之间需要搬运数据（显存带宽瓶颈，番外篇二）
  │
  ▼ 输出一个 token
  │
  ▼ 重复（第七篇：Decode 循环）
```

**大模型的全部计算 = 十几种算子 × 重复几百次。** 没有神秘的“AI 运算”——就是矩阵乘法、加法、取指数这些基础操作。让它显得“智能”的，不是单个算子有多复杂，而是第六篇训练出来的那些参数——它们决定了每个 MatMul 乘的是什么矩阵、每个 Embedding 查的是什么表。

-----

## 六、结语

算子是大模型计算的“原子操作”——不可再分的最小计算单位。

正篇十篇里你学到的每一个概念——Embedding、Attention、FFN、LayerNorm、残差连接——拆到执行层面，都是几个算子的组合。模型的全部“智能”，最终都要落地到这些朴素的计算步骤上。

> *大模型不会做任何“神奇”的运算。它做的每一步——查表、乘矩阵、取指数、做加法——你用计算器都能完成。只不过它在一秒钟内做了几千亿次。*
