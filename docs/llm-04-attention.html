04. Attention：大模型的“阅读重点”机制

Attention 让每个 token 在生成表示时，按权重查看前面的 token，从而把上下文关系带进当前表示。

AttentionQKVCausal Mask
读文章主旨 先看两张图

Thesis

文章主旨

Attention 让每个 token 在生成表示时，按权重查看前面的 token，从而把上下文关系带进当前表示。

一句话里每个词的重要性不同。模型生成最后一个词时，不能平均看前面所有词，而要知道当前最该关注谁。

Visual

两张图看懂

第一张图讲机制怎么流动，第二张图把抽象概念落到生活直觉。

Analogy

原文比喻

图书馆里按标签找书

文章把 Attention 比成在图书馆找书：先扫每本书的标签并打分，再把主要精力放到最相关的内容上。

对应到机制：当前 token 通过 Q 去匹配前文的 K，再按权重取走 V 中的信息。

Explain

概念拆解

按下面三步讲，会比直接抛术语更顺：先建立直觉，再解释机制，最后落到本页结论。

Step 01

先问问题

当前 token 会形成 Query，可以理解为“我现在需要找什么线索”。

Step 02

再匹配线索

前文 token 提供 Key 和 Value；Key 用来匹配，Value 是真正被取走的信息。

Step 03

最后加权汇总

Attention 权重不是平均分配，而是让相关位置贡献更多上下文信息。

本页要带走的三句话

  • Attention 负责在上下文里分配注意力权重。
  • 因果语言模型只能看当前位置及之前的 token。
  • 多头注意力让模型能同时从多个角度理解一句话。

Small Check

可选小实验

把抽象的 Attention 变成可观察的矩阵和热力图。 实验只用于观察现象，不作为本页主线。

可选实验命令

python3 llm/04-attention/attention.py "The capital of France is" --layer 0
  • Attention 矩阵会显示每个 token 对前面 token 的关注权重。
  • 右上角未来位置不可见，这是因果掩码的效果。
  • 最后一个 token 的注意力变化，可以直观看到模型如何汇总前文。
