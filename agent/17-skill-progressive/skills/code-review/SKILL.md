# 代码审查 Skill

## 审查流程
1. 用 execute_bash("find . -name '*.py' | head -20") 了解项目规模
2. 用 read_file 逐个读取关键文件
3. 按以下维度检查：
   - 安全性（SQL 注入、硬编码密钥）
   - 性能（N+1 查询、不必要的循环）
   - 可维护性（命名、注释、复杂度）
4. 输出结构化报告

## 报告格式
```
## 审查报告

### 严重问题
- [文件名:行号] 问题描述

### 建议改进
- [文件名:行号] 改进建议

### 总结
整体评价和优先级建议
```
